# Directory Listing Optimisation: A Checklist (with FormBeep Examples)

TL;DR
- Directories rarely “drive growth” by themselves, but they do three valuable things: backlinks, credibility, and incremental high-intent users.
- The fastest way to win is consistency: the same product story, same screenshots, same claims—everywhere.
- Your listing should answer one question immediately: “What happens after I install this?”
- For FormBeep, that outcome is simple: a form submission triggers an instant WhatsApp alert — no Zapier required (email/SMS planned).

## The problem (framing)
Most directory submissions fail for boring reasons:
- Unclear tagline (sounds like everything).
- Feature list that doesn’t map to a real use case.
- No screenshots (or screenshots that don’t prove the outcome).
- Claims that don’t match the actual product.

That last one is the biggest risk for an early-stage product. If you’re new and you only have a small number of customers, trust is fragile.

So the goal is not “be hype”. The goal is “be specific and provable”.

## Who it’s for
- Indie hackers and founders doing distribution work themselves.
- Agencies listing client tools (or your own internal tools).
- Anyone preparing a Product Hunt launch or building a directory backlink baseline.

## Step-by-step directory optimisation checklist
### Step 1: Lock a single sentence that’s actually true
A good one-liner:
- Names the trigger
- Names the destination
- Removes complexity

FormBeep (truthful version):
- “Instant WhatsApp alerts for your website forms — no Zapier required.”

If you want to mention additional channels (email/SMS), do it as roadmap unless verified on the live product:
- “WhatsApp-first form alerts (email/SMS planned) — no Zapier required.”

### Step 2: Write a “what happens next” description
People skim directory listings. Use a short paragraph that describes the flow:

Template:
- “Connect your existing form. When someone submits it, you instantly receive a message with the lead details. Test it once. Done.”

FormBeep example paragraph:
- “FormBeep connects your website forms to WhatsApp notifications. When someone submits your contact/booking/lead form, you get an alert in seconds—so you can reply while the lead is still warm. Setup is a script tag + WhatsApp verification, with a free tier for testing.”

### Step 3: Prepare a screenshot set that proves the outcome
You want screenshots that answer:
1) Where do I configure it?
2) What does the alert look like?
3) Does it work with my stack?

Recommended 3–5 screenshots:
- Dashboard “Add your site / API key”
- WhatsApp verification (QR step)
- Example WhatsApp notification
- Integrations list (Webflow/WordPress/Framer/custom)
- Pricing snippet showing free tier limit (15 notifications/month)

### Step 4: Make your categories and tags match how people search
Rule: your primary category should match the buyer’s job-to-be-done.

For FormBeep, strong options are:
- Lead generation
- Form notifications
- Sales enablement (if the directory supports it)

Tag examples (choose 5–10):
- WhatsApp
- form notifications
- lead alerts
- Webflow
- WordPress
- Framer
- webhook
- Zapier alternative

### Step 5: Use “founder comment” space to tell a tight origin story
Many directories (especially Product Hunt) reward a human reason.

Structure:
- What problem you saw
- Why existing tools were overkill
- What you built instead

FormBeep-style founder comment (template):
- “I built this because clients kept missing leads in email. Automation tools work, but they’re too much for a simple alert. FormBeep sends form submissions straight to WhatsApp so teams can respond immediately.”

### Step 6: Use consistent URLs and internal tracking
For each directory submission:
- Link to the homepage or a specific landing page.
- Add UTM parameters so you can see directory performance later.

Example:
- https://formbeep.com/?utm_source=producthunt&utm_medium=directory&utm_campaign=launch

### Step 7: Don’t overclaim (especially on channels)
If your live product is WhatsApp-first, keep the listing WhatsApp-first.

Safe language patterns:
- “WhatsApp alerts today”
- “SMS/email planned” (only if that’s true and you’re comfortable committing)

Avoid:
- “Official integration” badges if you don’t have a marketplace app.
- “One-click install” unless it’s literally one click.

## Directory submission workflow (copy/paste)
Use this exact workflow to avoid inconsistency:

1) Create a “directory profile” doc with:
- one-liner
- short description (150 chars)
- long description (500 chars)
- tags
- categories
- pricing URL
- docs URL
- support email

2) Export your screenshot pack once.

3) Submit to 5–10 directories in one batch.

4) Track:
- submission date
- status
- listing URL
