# FormBeep — WordPress plugin post-approval launch plan

Owner: Moxie (plan) + Rishi (execution on WP.org account)
Status: WAITING FOR WP APPROVAL

## Objective
Turn WP.org approval into (1) installs, (2) traffic to FormBeep site, (3) activated accounts, (4) early paid conversions.

## Day 0 (same day approval lands)
1) WP.org listing hygiene
- Ensure: short description, tags, banners/icons, screenshots, FAQ, and “Support” link are correct.
- Add 2–3 “Why this exists” bullets aimed at the primary use-case: Form → WhatsApp/SMS/email alerts without Zapier.
- Verify: tested up to current WP version, PHP version support.

2) Trackable links
- Add UTM’d links from WP.org “Plugin Homepage” and/or FAQ to FormBeep money pages.
- Create a dedicated landing page on FormBeep: `/integrations/wordpress/` (or similar) that matches WP listing copy and captures intent.

3) First distribution burst
- Announce on:
  - FormBeep site blog (short post)
  - X / LinkedIn (if used)
  - Relevant communities (careful: not spam)

## Week 1
1) Support-driven growth
- Reply fast to WP.org support threads.
- Maintain a public “Known issues” section (can be in docs) to reduce churn.

2) Content assets (SEO + conversion)
- Publish 2 high-intent pages:
  - “WordPress form to WhatsApp notifications”
  - “WordPress form to SMS notifications”
- Add comparison/alternative page:
  - “FormBeep vs Web2Phone (WordPress)”

3) Directory & partner flywheel
- Add “WordPress plugin available” badge/line to every directory submission.
- Reach out to a small list of WP agencies/builders: offer them a setup guide + affiliate/discount.

## Week 2–4
1) Iterate listing conversion
- If impressions are high but installs low: adjust title, first 2 sentences, and screenshot order.
- If installs are high but activations low: improve onboarding + in-plugin UX copy (Rishi-owned).

2) Add docs + video
- 2-minute setup video.
- Troubleshooting doc.

## KPIs to watch
- WP.org: installs (active installs proxy), ratings, support thread resolution time.
- FormBeep: visits to /integrations/wordpress/, signup clicks, activation rate from WP channel.

## Dependencies / constraints
- Plugin code changes are founder-owned. Automation must not modify plugin code or submission.
