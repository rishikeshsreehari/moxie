# Landing optimization loop (low-traffic friendly)

Owner mapping
- Forge: technical CRO + instrumentation plan (events, tracking)
- Mira: analytics readouts + weekly deltas
- Kiro: above-the-fold copy + page narrative tweaks

## Reality check (10 uniques/day)
- A/B testing is not statistically viable.
- But qualitative improvements ARE viable using:
  - session recordings (Microsoft Clarity)
  - event counts (Umami events)
  - copy clarity checks (5-second test)

## Phase 1 — Instrument (same day)
Track these events on formbeep.com landing:
- hero_primary_cta_click (e.g., “Get started”)
- hero_secondary_cta_click (e.g., “See demo” / “Pricing”)
- pricing_click
- docs_click
- integrations_click
- scroll_25 / scroll_50 / scroll_75

Optional (recommended): Microsoft Clarity for recordings + rage clicks.

## Phase 2 — Baseline (3–7 days)
Collect:
- total visitors
- clicks per CTA (absolute + % of visitors)
- top referrers/UTMs
- 10–30 recordings reviewed

## Phase 3 — Make 1 high-confidence change (per week)
Examples that don’t need high traffic:
- tighten title/subtitle to 1-sentence clarity
- add 1 product-in-action visual (GIF)
- add 1 proof element (testimonial / “used by X” / founder note)
- reduce friction: CTA -> value-based (“Get WhatsApp alerts”) + objection handle (“No Zapier”) 

## Phase 4 — Review
After 7 days:
- did hero CTA click rate increase?
- did pricing clicks increase?
- did signups increase (even if absolute is small)?

Rule: one change at a time + log what changed.
