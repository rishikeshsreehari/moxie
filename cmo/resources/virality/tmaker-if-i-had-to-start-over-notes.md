# Notes — tmaker: “If I had to start over” (what we should copy)

Captured: 2026-04-01

## What I’m copying into our org (FormBeep + future products)

### 1) Distribution is the product (daily reps)
Adopt: we run a daily “distribution block” that is non-negotiable.
- 2 directory submissions/day (manual)
- 3 comments/day (Reddit / IndieHackers / communities)
- 1 outreach touch/day (agency/platform)

Why: these are compounding loops; they create backlinks, discovery, and “language from the market.”

### 2) Ship small, frequently, and narrate it
Adopt: every week we publish:
- 1 shipping note (changelog / feature) + 1 story post (why/lesson)
Then cross-post snippets into 2–3 places (where allowed).

Why: “shipping + narrating” creates repeated surface area without feeling spammy.

### 3) Treat content as an asset library, not one-off posts
Adopt: each post must have:
- internal links to 2–3 existing posts/pages
- an image placeholder (so we can later add visuals)
- a reusable excerpt (for directory descriptions, social, FAQs)

### 4) Prove conversion, then scale discovery
Adopt: delay Product Hunt / big launches until we have:
- crisp demo
- 1–2 testimonials or credible proof
- high-converting landing page paths

### 5) Build “loops” not “campaigns”
Adopt loops we can run forever:
- directories → backlink → GSC impressions → update pages
- Reddit comments → objections → FAQ updates → higher conversion
- agency outreach → integration pages → more directories/blog hooks

I should not tell you “go follow this.”
My job is to translate it into 2–3 repeatable weekly rules, keep them in HQ resources, and assign work to employees to execute.
