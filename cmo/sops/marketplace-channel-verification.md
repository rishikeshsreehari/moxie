# SOP — Marketplace/Platform Channels: Last-mile Verification (before planning)

Purpose: prevent wasted cycles from mis-scoped “submission” work when platforms actually require full integrations/apps.

## Rule
No one may claim “ready to submit” for a platform marketplace until the checklist below is completed with evidence.

## Checklist (must be completed per platform)
1) Platform type
- [ ] Directory listing (metadata-only)
- [ ] Marketplace app/plugin (requires build + review)

2) Required artifact(s)
- [ ] Integration type (plugin / SDK / component / OAuth app / webhook receiver)
- [ ] Required auth (OAuth, API key, none)
- [ ] Required docs URLs (developer docs + submission guidelines)

3) Evidence (attach or quote)
- [ ] Screenshot or quoted text from the platform submission page stating requirements
- [ ] Link to guidelines section on “what qualifies” (e.g., Webflow app vs template vs library)

4) MVP scope
- [ ] Minimal viable integration definition
- [ ] Rough eng effort (S/M/L)
- [ ] “Build vs defer” recommendation

5) Submission readiness definition
- [ ] What “ready to submit” means for this platform (exact conditions)

## Output format
A 1-page matrix: one row per platform with the above filled out + a clear YES/NO for “can submit today”.
