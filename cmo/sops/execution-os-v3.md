# Execution OS v3 (Founder + Moxie)

Purpose: convert “plans” into “executed actions” daily, with minimal founder rework.

## Daily Founder Execution Window
- Default: 13:00 GST (UTC+4) = 09:00 UTC
- Rule: we can shift per-platform if the platform’s optimal window conflicts.

## The Last-Mile Contract (Definition of Done)
A task is not DONE unless it ends in an action that can be executed in <10 minutes by the owner.

Examples:
- Reddit: DONE = 1 paste-ready post packet for a specific subreddit + posting window + rules compliance + comment #1.
- X: DONE = 1 post + 3 replies (targets listed) + posting window + why-now.
- Directories: DONE = verified submission URL + requirements verified timestamp + copy bundle + what to screenshot/log.
- Blog: DONE = publish-ready markdown (frontmatter OK, internal links, CTA blocks) + exact deploy steps.

## Content Calendar Golden Rule (Hard requirement)
Every planned post must include:
1) WHERE
2) WHY (intent + KPI)
3) HOW (paste-ready packet + assets)
4) WHEN (exact UTC time + target country/timezone)
5) WHY-NOW (evidence-based, or explicitly marked hypothesis)

## Two-Queue System
1) Research Queue: messy allowed; cannot be shown as execution-ready.
2) Execution Queue: must satisfy the Last-Mile Contract; only this queue feeds the 13:00 GST window.

## QA gates
- Founder-Ready Gate applies BEFORE anything reaches Rishi.
- Any deliverable that fails gets a REWORK task immediately (same day).

## Completion handshake (hard requirement)
A task is not complete unless:
1) Moxie reports it to Rishi (with file/link)
2) Rishi explicitly marks it DONE
This prevents silent “done” states and missed deliverables.
## Operating cadence
- By 12:45 GST: Moxie delivers 1–3 paste-ready packets to /root/moxie_hq/cmo/rishi_review.md under “TODAY 13:00 GST”.
- At 13:00 GST: Rishi executes 1 packet (or 2 if trivial) and reports result (URL, screenshot, approval/denial).
- After execution: Moxie logs outcome and updates the channel log + next step.

## Metrics to track (minimum)
Per product, daily:
- Visits (Umami)
- Primary intent event clicks (CTA)
- Free signups (manual + instrumented)
- Paid conversions (manual)

## Escalation
If we miss 2 consecutive execution windows without a posted/submitted/published outcome, Moxie must produce a 1-page root-cause + fix plan before proceeding.
